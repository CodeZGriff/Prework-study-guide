Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • !DOCTYPE is not case sensensitive because it is a declaration. Declarations are non case-sensitive; HTML elements are case-senstive
  • HTML attributes are defined in the beginning element, never the closing element
  • Users don't see info in head or meta elements
  • Child elements are elements nested inside other elements. The holder of the child elements is the parent element
  • Heading tags should be in order from h1 to h6. H1 should only appear once as the title
  • Alt text can improve search engine rankings on website

CSS

  • Seperation of concerns-refers to separating code into different files based on use. Each section of code should have its own responsibility.
  • Declarations- 2 important components - CSS property we want to apply and the value of the property
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Block Value- assigns each image to start on a new line

Git

  • Contains the tools to set up repositories
  • You can create branches to work on code outside of the main branch
  • Branch changes must be approved before being merged with the main section

JavaScript

  • Javascript makes webpage interactive
  • Variables are containers that store values. You start by declaring a variable with the let keyword, followed by the name you give to the variable
  • JavaScript is case sensitive. This means myVariable is not the same as myvariable. If you have problems in your code, check the case!
  • String - sequence of text that is enclosed in single or double quote marks
  • Number - this is a number and doesn't have quotes around them
  • Boolean - True/false value - special keywords that don't need quote marks
  • Array - structure that allows you to store multiple values in a single reference
  • Object - can be anything, everything in Javascript is an object and can be stored in a variable
  • Comments - snippets of text that are added along with code, but browser ignores text marked as comments
  • Events - code structures that listen for activity, most common is the click event, which is fired when clicking with the mouse
  • All primitive types, except NULL and UNDEFINED, have their corresponding object wrapper types, which provide useful methods for working with the primitive values
